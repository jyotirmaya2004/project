Media Queries — Types & Practical Examples

Resize the window, open print preview, or emulate devices to see each media query in action.

This single-file demo shows screen, print, speech, all, plus examples of orientation & resolution.

Media Type: screen

Used for visual screens (desktops, phones, tablets). Most responsive CSS uses @media screen (or omits screen since it's the default).

Resize the window:
Breakpoint: max-width:900px → stacked layout Orientation: landscape (min-width:700px style) Resolution: min-resolution:2dppx → HiDPI badge

Try developer tools to emulate mobile/desktop and rotate to see orientation rules.

Media Type: print

Applies when printing or in print preview. Use it to simplify pages: hide navigation, change colors to black/white, adjust fonts and pagination.

  1. Click the Print Preview button above (or press Ctrl+P).
  2. Notice how background, navigation hints and buttons disappear.
  3. Paper-friendly styles are in the CSS under @media print.

Media Type: speech

Intended for speech synthesizers (screen readers with style support). Browser support is limited — this is for educational completeness.

If your user agent supports CSS speech properties, this media query allows you to change voice, pitch, rate, etc. Most modern screen readers ignore CSS speech rules; use ARIA and semantic HTML for accessibility instead.

Media Type: all (universal)

The all media type matches every device. It's equivalent to omitting the media type. Useful when you want a conditional rule to apply everywhere beyond a certain condition.

This demo adds extra page padding when the viewport is wide (see the @media all and (min-width:1200px) rule in the CSS).

Deprecated / Historical Media Types

Older CSS specs listed types like tv, projection, handheld, tty, braille, embossed, and aural. Modern browsers generally ignore them. Use feature queries and responsive design instead.

This demo contains commented examples of such media types in the CSS for reference (not active in most browsers).

Tips: use DevTools to test device emulation, rotate device, toggle print preview. For accessibility, prefer semantic HTML + ARIA instead of relying solely on @media speech.
